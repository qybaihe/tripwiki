# 省直辖县级行政区划（异常行政聚合桶说明）

> 状态：不作为单一 TripWiki 城市攻略编写对象
>
> 口径说明：本条对应多个省级数据源中的 `省直辖县级行政区划` 聚合项，仓库已统一标记为 `skipped`。

## 1. 结论

“省直辖县级行政区划”不是一个真实、单一、可独立成篇的旅游目的地，而是行政区划整理时常见的**异常聚合桶名称**。它通常只是把若干由省级直接代管、代办或统计归并的县级市/县/林区/自治县临时装进一个总类目里，**并不对应一个可以被游客实际抵达、停留、游玩的单体城市**。

因此：

- **不应把 `省直辖县级行政区划.md` 写成常规城市攻略**；
- 它更适合保留为**说明型占位页**；
- 真正可用的旅行内容，应当分流到对应的真实目的地页面。

这也与仓库现有总口径一致：
- `README.md` 已明确：异常聚合行政桶可以 `skipped`；
- `data/progress.json` 中，河南省、湖北省、海南省对应的 `省直辖县级行政区划` 均已标记为 `skipped`；
- 备注均为：`abnormal administrative bucket skipped by coordinator`。

## 2. 为什么不能按常规城市攻略写

TripWiki 的城市攻略对象，原则上应满足以下至少几项：
- 有明确、唯一的地理边界与旅行对象；
- 有唯一或高度稳定的城市入口与交通组织；
- 能形成同一目的地下的景点、住宿、美食、行程与避坑逻辑；
- 旅行者能直接把页面标题当成搜索与出行对象。

而“省直辖县级行政区划”不满足这些条件： 

1. **没有唯一目的地**：这个名称在不同省份指向完全不同的一组县级行政单位；
2. **没有唯一交通入口**：济源、仙桃、儋州、神农架林区等出行方式完全不同；
3. **没有统一景点体系**：海南岛海滨度假、湖北江汉平原城市、豫西山地城市，不能被合并成一个攻略；
4. **没有统一住宿/美食/行程逻辑**：分布跨度大，硬写只会变成失真拼盘。

如果把它写成普通旅游页，结果只会是把多个真实目的地胡乱打包，既不准确，也不利于读者落地执行。

## 3. 应该分流到哪些真实目的地

### 3.1 河南省对应分流
河南这里实际应分流到：
- `cities/济源.md`

也就是说，若数据源出现“河南省::省直辖县级行政区划”，旅行语义上通常不该继续停留在这个桶，而应直接查看**济源**页面。

### 3.2 湖北省对应分流
湖北这里实际应分流到：
- `cities/仙桃.md`
- `cities/潜江.md`
- `cities/天门.md`
- `cities/神农架林区.md`

这几项的性质本身就不同：
- 仙桃、潜江、天门更接近独立县级市旅行对象；
- 神农架林区则是强识别度的山地生态型目的地；
- 把它们统称成一个“省直辖县级行政区划”页面，没有实际旅行意义。

### 3.3 海南省对应分流
海南这里实际应分流到一组真实、已单列的县级/县级市目的地，包括：
- `cities/儋州.md`
- `cities/五指山.md`
- `cities/文昌.md`
- `cities/琼海.md`
- `cities/万宁.md`
- `cities/东方.md`
- `cities/定安.md`
- `cities/屯昌.md`
- `cities/澄迈.md`
- `cities/临高.md`
- `cities/白沙黎族自治县.md`
- `cities/昌江黎族自治县.md`
- `cities/乐东黎族自治县.md`
- `cities/陵水黎族自治县.md`
- `cities/保亭黎族苗族自治县.md`
- `cities/琼中黎族苗族自治县.md`

这些页面才是游客真正应该进入的对象。比如：
- 看航天与滨海生活，去文昌；
- 看海湾度假，去万宁；
- 看热带雨林与山地气候，去五指山或保亭；
- 看火山、海花岛周边与西部滨海线，可看儋州或东方。

## 4. 本页的正确用途

本页存在的意义不是提供旅游内容，而是：
- **阻止后续协作误把异常桶当成正常城市继续扩写**；
- 明确告诉编辑者与读者：这里需要**分流**，不是**合并编写**；
- 作为 `progress.json` 中 `skipped` 状态的文字化解释页。

## 5. 行程 / 景点 / 美食 / 住宿 / 交通

本条目**均不适用**。

原因很简单：
- 它不对应一个城市；
- 也不对应一个自治州或单一区域；
- 无法产出可信的统一 1—3 天路线；
- 无法形成统一景点池、住宿区或餐饮建议。

请直接进入对应真实页面查看具体内容。

## 6. 避坑

- **不要把“省直辖县级行政区划”当成一个真实城市名使用**；
- **不要据此制定行程**；
- 如果在数据清洗、任务分发或文件生成中再次见到该名称，应优先判断为行政聚合桶；
- 后续若继续补库，应补具体城市，不应继续往这个桶里拼内容。

## 8. 最终处理意见

`cities/省直辖县级行政区划.md` 应保留为**异常聚合桶说明页**，不写成常规攻略。

后续正确做法是：
- 在协作与任务分配中继续将其视为 `skipped`；
- 用户或编辑若看到该名称，应按所在省份分流到相应真实目的地；
- 不再把它当作一个需要补齐景点、美食、住宿章节的正常城市页。
